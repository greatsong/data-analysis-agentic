    Ch.5 --- 분석 지침서(Soul Document) 작성 가이드¶

    Part 2: AI에게 분석을 시켜라 | 심화 자료

    이 장의 목적

    이 장은 Ch.4에서 소개한 Soul Document의 5개 섹션을 깊이 있게 다룹니다. 각 섹션의 작성 요령, 좋은 예시와 나쁜 예시, 교사 코멘트를 제공하며, 인터랙티브 차트를 통해 데이터 분석의 핵심 개념을 체험할 수 있습니다.

    1. Soul Document란?¶

    개념 정의¶

    Soul Document는 AI에게 데이터 분석을 지시하기 위한 구조화된 문서입니다. 단순히 "분석해줘"라고 하는 대신, 분석의 목적, 데이터, 방법, 시각화, 해석 기준을 체계적으로 작성합니다.

    왜 Soul Document인가?

    • Soul = 분석의 영혼, 핵심 의도
    • Document = 문서화된 지침
    • 즉, "이 분석이 왜 필요하고, 무엇을 어떻게 분석하고, 결과를 어떻게 판단할지"를 분석의 영혼까지 담은 문서입니다.

    Soul Document 전체 구조 미리보기¶

    Soul Document 예시: 학년별 수면시간 분석

    1. 배경 맥락 이 분석은 우리 학교 학생들의 수면 패턴을 파악하기 위한 것입니다. 특히 학년에 따라 수면시간에 차이가 있는지 확인하여, 학년별 맞춤 수면 교육 프로그램의 필요성을 판단하고자 합니다.

    2. 데이터 설명 데이터: school_survey.csv (200명, 12개 변수) 주요 변수: grade(학년 1/2/3), sleep_weekday(평일 수면시간, 시간), sleep_weekend(주말 수면시간), fatigue(피로도 1~5)

    3. 분석 방법 (1) 학년별 기술통계(평균, 표준편차, 중앙값) 계산 (2) 학년 간 차이에 대해 일원분산분석(ANOVA) 검정 (3) 유의수준 0.05 기준 p-value 보고

    4. 시각화 요구사항 (1) 학년별 수면시간 박스플롯 (한국어 라벨) (2) x축: 학년, y축: 평균 수면시간(h) (3) 학년별 색상 구분, 그래프 제목 포함

    5. 해석 기준 (1) p < 0.05이면 학년 간 차이 유의 (2) 평균 차이 30분 이상이면 실질적 의미 있음 (3) 한 문단으로 결론 요약, 한계점 명시

    2. 섹션별 작성 가이드¶

    섹션 1: 배경 맥락 --- "왜 이 분석을 하는가"¶

    작성 요령¶

    배경 맥락은 AI에게 분석의 방향을 잡아주는 나침반입니다. 이 섹션이 없으면 AI는 데이터에서 아무 패턴이나 찾으려 합니다.

    포함해야 할 요소:

    1. 분석의 동기: 어떤 문제나 궁금증에서 시작했는가
    2. 분석의 목적: 결과를 어디에 활용할 것인가
    3. 대상 독자: 누구를 위한 분석인가

    좋은 예시 vs 나쁜 예시¶

    나쁜 예시

    "수면 데이터를 분석해줘."

    문제점:

    • 동기 없음: 왜 수면 데이터?
    • 목적 없음: 결과로 뭘 하려고?
    • 방향 없음: 뭘 찾아야 하는지 불명확

    좋은 예시

    "우리 학교에서 1교시 수업 집중도가 학년마다 다르다는 교사 관찰이 있습니다. 수면 부족이 원인일 수 있다는 가설하에, 학년별 수면시간 차이를 확인하고자 합니다. 결과는 학년별 조회 시간 조정 건의에 활용할 예정입니다."

    강점:

    • 동기 명확: 교사 관찰에서 출발
    • 목적 구체: 조회 시간 조정 건의
    • 가설 제시: AI에게 분석 방향 안내

    교사 코멘트

    배경 맥락이 구체적일수록 AI의 분석도 초점이 맞습니다. 학생들에게 "분석 결과를 가지고 누구에게 뭘 건의할 것인지" 먼저 생각하게 하면, 배경 맥락 작성이 훨씬 수월해집니다.

    섹션 2: 데이터 설명 --- "어떤 데이터를 분석하는가"¶

    작성 요령¶

    AI는 데이터 파일을 받아도 맥락을 모릅니다. "grade"라는 컬럼이 학년인지 성적인지 알려줘야 합니다.

    포함해야 할 요소:

    1. 파일명과 형식: CSV, Excel 등
    2. 데이터 규모: 행 수(관측치), 열 수(변수)
    3. 주요 변수 설명: 변수명, 의미, 단위, 척도
    4. 특이사항: 결측값, 이상치, 코딩 규칙

    좋은 예시 vs 나쁜 예시¶

    나쁜 예시

    "첨부 파일 참고."

    문제점:

    • AI가 변수명을 추측해야 함
    • 단위를 모르면 해석 오류 발생
    • 결측값 처리 방법을 알 수 없음

    좋은 예시

    "파일: school_survey.csv (200행 x 12열) 주요 변수: - grade: 학년 (1, 2, 3) - sleep_weekday: 평일 수면시간 (시간, 소수점 1자리) - fatigue: 피로도 (1=매우 낮음 ~ 5=매우 높음) - stress: 스트레스 (1~10, 연속형)

    결측값: 약 5% 존재, 분석 시 제거(listwise deletion)"

    강점:

    • 변수마다 의미, 단위, 척도 명시
    • 결측값 처리 방법 사전 지정
    • AI가 추측할 필요 없음

    교사 코멘트

    데이터 설명은 지루하지만 가장 중요한 섹션입니다. 이 섹션이 부실하면 AI가 변수를 잘못 해석하여 전체 분석이 틀어집니다. "귀찮아도 변수 하나하나 설명하세요"라고 강조하세요.

    섹션 3: 분석 방법 --- "어떻게 분석하라"¶

    작성 요령¶

    AI에게 "알아서 해줘"라고 하면 AI는 가장 기본적인(그리고 종종 부적절한) 방법을 선택합니다. 분석 방법을 지정하면 더 정확한 결과를 얻을 수 있습니다.

    포함해야 할 요소:

    1. 기술통계: 어떤 요약 통계량을 계산할지
    2. 추론통계: 어떤 검정/모델을 사용할지
    3. 순서: 분석을 어떤 순서로 할지
    4. 유의수준: 통계적 판단의 기준

    데이터 유형별 분석 방법 매칭표¶

    이 표를 학생에게 참고자료로 제공하세요

    학생들은 어떤 분석 방법을 써야 할지 모르는 경우가 많습니다. 아래 매칭표를 보고 자신의 질문에 맞는 방법을 선택할 수 있습니다.

    데이터 유형 질문 유형 추천 분석 방법 추천 시각화
    두 연속변수 "A가 커지면 B도 커지는가?" 상관분석, 회귀분석 산점도
    범주 x 연속 "그룹 A와 B의 평균이 다른가?" t검정(2그룹), ANOVA(3+그룹) 박스플롯, 막대그래프
    시계열 "시간에 따라 변하는가?" 추세분석, 이동평균 선그래프
    여러 변수 관계 "변수들 간 관계가 어떤가?" 상관행렬 히트맵
    범주 x 범주 "두 범주 간 연관이 있는가?" 카이제곱 검정 모자이크 플롯
    비율 비교 "이 비율이 기대와 다른가?" 이항검정, 비율검정 파이차트, 막대그래프

    좋은 예시 vs 나쁜 예시¶

    나쁜 예시

    "통계 분석을 해줘."

    문제점:

    • 어떤 통계? 기술? 추론?
    • AI가 임의로 방법 선택
    • 결과의 의미를 해석할 수 없음

    좋은 예시

    "분석 순서: 1) 학년별 기술통계: 평균, 표준편차, 중앙값, 최솟값, 최댓값 2) 정규성 검정: Shapiro-Wilk (각 학년별) 3) 등분산 검정: Levene's test 4) 학년 간 비교: 정규성 충족 시 ANOVA, 미충족 시 Kruskal-Wallis 5) 사후검정: Tukey HSD (ANOVA인 경우) 6) 유의수준: 0.05"

    강점:

    • 단계별 순서 명확
    • 조건부 방법 지정
    • 유의수준 사전 설정

    교사 코멘트

    학생 수준에 따라 분석 방법의 상세도를 조절하세요. 초보 학생은 "평균 비교 → 박스플롯" 수준으로도 충분합니다. 상위 학생은 검정 방법까지 지정하도록 유도할 수 있습니다.

    섹션 4: 시각화 요구사항 --- "어떤 그래프를 그려라"¶

    작성 요령¶

    시각화를 지정하지 않으면 AI는 기본 설정의 투박한 그래프를 생성합니다. 좋은 시각화 지시는 그래프 유형, 축, 색상, 라벨을 모두 포함합니다.

    포함해야 할 요소:

    1. 그래프 유형: 산점도, 막대그래프, 박스플롯, 히트맵 등
    2. 축 설정: x축, y축 변수와 라벨
    3. 색상/스타일: 그룹별 색상, 글꼴 크기 등
    4. 제목과 범례: 그래프 제목, 범례 위치
    5. 추가 요소: 추세선, 평균선, 신뢰구간 등

    좋은 예시 vs 나쁜 예시¶

    나쁜 예시

    "그래프 그려줘."

    문제점:

    • 어떤 그래프? 어떤 변수?
    • AI가 아무 그래프나 생성
    • 라벨이 영어, 크기가 작음

    좋은 예시

    "시각화 요구사항: 1) 학년별 수면시간 박스플롯 - x축: '학년' (1학년/2학년/3학년) - y축: '평균 수면시간(h)' (범위: 4~10) - 색상: 1학년=파랑, 2학년=초록, 3학년=빨강 - 제목: '학년별 평일 수면시간 분포' - 개별 데이터 포인트를 점으로 표시 2) 수면시간 vs 피로도 산점도 - 추세선 포함, R-squared 표시 - 모든 라벨 한국어"

    강점:

    • 그래프 유형 + 변수 지정
    • 축 라벨, 색상, 제목 모두 포함
    • 한국어 라벨 명시

    섹션 5: 해석 기준 --- "결과를 어떤 기준으로 판단하라"¶

    작성 요령¶

    AI는 숫자를 보고하지만, 그 숫자가 의미 있는지는 기준이 있어야 판단할 수 있습니다. 해석 기준을 미리 정해두면 결과를 일관되게 해석할 수 있습니다.

    포함해야 할 요소:

    1. 통계적 유의성 기준: p-value 임계값
    2. 실질적 유의성 기준: 차이가 "의미 있으려면" 얼마나 커야 하는가
    3. 해석 형식: 결론을 어떤 형태로 작성할 것인가
    4. 주의사항: 인과 추론 금지, 한계점 명시 등

    좋은 예시 vs 나쁜 예시¶

    나쁜 예시

    "결과를 해석해줘."

    문제점:

    • AI가 "상관관계가 있습니다"라고만 말함
    • 수치적 기준 없이 주관적 해석
    • 한계점 무시

    좋은 예시

    "해석 기준: 1) 통계적 유의성: p < 0.05 → 차이 유의, p >= 0.05 → 차이 없음 2) 실질적 유의성: 평균 차이 30분 이상 → 의미 있는 차이 3) 효과 크기: Cohen's d 보고 (0.2=작음, 0.5=중간, 0.8=큼) 4) 결론 형식: 한 문단 요약 + 근거 수치 인용 5) 주의사항: '이 결과는 상관관계이며, 인과관계를 의미하지 않습니다' 포함 6) 한계점: 표본의 대표성, 자기보고 편향 등 명시"

    강점:

    • 이중 기준 (통계적 + 실질적)
    • 결론 형식 지정
    • 한계점 요구

    교사 코멘트

    해석 기준 섹션은 비판적 사고를 가장 잘 반영하는 부분입니다. "p-value만 보면 되는 거 아니에요?"라는 질문이 나오면, "p-value가 작아도 차이가 5분뿐이면 의미가 있을까?"라고 되물어보세요. 통계적 유의성과 실질적 유의성의 차이를 깨닫는 순간이 됩니다.

    4. 이상치와 결측값 처리 지침¶

    Soul Document에 이상치 처리를 포함하는 방법¶

    이상치와 결측값은 분석 결과를 크게 좌우합니다. Soul Document에 처리 방법을 명시하면 AI가 일관되게 처리합니다.

    결측값 처리 옵션¶

    방법 설명 Soul Document 작성 예
    제거(Listwise) 결측값이 있는 행 전체 삭제 "결측값이 있는 행은 분석에서 제외하세요"
    평균 대체 해당 변수의 평균으로 대체 "결측값은 해당 변수의 평균으로 대체하세요"
    중앙값 대체 해당 변수의 중앙값으로 대체 "결측값은 중앙값으로 대체하세요 (이상치 영향 최소화)"
    보고만 결측값 현황만 보고 "결측값 현황(변수별 개수, 비율)을 먼저 보고하세요"

    학생 수준별 추천

    • 초보: "결측값이 있는 행은 제외하세요" (가장 단순)
    • 중급: "결측값 현황을 보고한 후, 제거하세요"
    • 상급: "결측값 비율이 5% 미만이면 제거, 이상이면 평균 대체"

    이상치 처리 옵션¶

    방법 설명 Soul Document 작성 예
    IQR 방법 Q1-1.5xIQR ~ Q3+1.5xIQR 범위 밖 제거 "IQR 방법으로 이상치를 탐지하고 제거하세요"
    Z-score 평균에서 3 표준편차 이상 벗어난 값 "Z-score 3 이상인 값을 이상치로 표시하세요"
    보고만 이상치 목록만 보고 "이상치를 탐지하되 제거하지 말고, 포함/제외 두 결과를 비교하세요"

    5. 실전 연습: 완전한 Soul Document 예시¶

    예시 1: 수면시간과 피로도의 관계¶

    Soul Document: 수면시간과 피로도의 관계 분석

    1. 배경 맥락

    우리 학교 보건교사가 최근 보건실 방문 학생 중 "피곤하다"고 호소하는 비율이 증가했다고 보고했습니다. 수면 부족이 원인인지 확인하기 위해, 수면시간과 피로도의 관계를 분석하고자 합니다. 분석 결과는 "적정 수면시간 가이드라인" 제작에 활용할 예정입니다.

    2. 데이터 설명

    • 파일: school_survey.csv (200행 x 12열)
    • 분석 대상 변수:
      • sleep_weekday: 평일 평균 수면시간 (시간, 연속형, 소수점 1자리)
      • fatigue: 피로도 (1=매우 낮음 ~ 5=매우 높음, 순서형)
      • grade: 학년 (1, 2, 3, 범주형) --- 보조 변수
    • 결측값: 약 5% 존재, 해당 행 제거(listwise deletion)

    3. 분석 방법

    1) 전체 기술통계: sleep_weekday, fatigue의 평균, 표준편차, 중앙값 2) 상관분석: sleep_weekday와 fatigue의 Pearson 상관계수 계산 3) 학년별 상관분석: 학년을 나누어 각각의 상관계수 비교 4) 단순 회귀분석: sleep_weekday → fatigue (예측 모델) 5) 이상치 탐지: IQR 방법으로 수면시간 이상치 확인

    4. 시각화 요구사항

    1) 수면시간 vs 피로도 산점도 - x축: '평일 수면시간(h)', y축: '피로도(점)' - 학년별 색상 구분 (1학년=파랑, 2학년=초록, 3학년=빨강) - 전체 추세선(검정 점선) + R-squared 표시 - 제목: '수면시간과 피로도의 관계' 2) 학년별 수면시간 박스플롯 - 개별 데이터 포인트 표시, 한국어 라벨 3) 모든 텍스트 한국어, 글꼴 크기 12pt 이상

    5. 해석 기준

    1) 상관계수: |r| > 0.3이면 의미 있는 상관, |r| > 0.5이면 강한 상관 2) p-value: < 0.05이면 통계적으로 유의 3) 결론 형식: 3문장 이내 요약 + 근거 수치 인용 4) "이 결과는 상관관계이며, 수면시간이 피로도를 직접 유발한다는 인과적 해석은 불가합니다" 포함 5) 한계점: 자기보고식 설문의 편향 가능성, 교란 변수(운동량, 카페인 섭취 등) 미통제 명시

    예시 2: 요일별 급식 만족도 비교¶

    Soul Document: 요일별 급식 만족도 분석

    1. 배경 맥락

    학생회에서 "금요일 급식이 월요일보다 맛없다"는 민원이 있었습니다. 요일에 따라 급식 만족도에 실제로 차이가 있는지 데이터로 확인하여, 영양사 선생님과의 간담회 자료로 활용하고자 합니다.

    2. 데이터 설명

    • 파일: lunch_survey.csv (150행 x 5열)
    • 변수: student_id, weekday(월~금), satisfaction(1~5점), menu_type(한식/양식/중식), grade
    • 결측값: 없음 (사전 정제 완료)

    3. 분석 방법

    1) 요일별 만족도 기술통계 (평균, 표준편차) 2) 요일 간 차이: ANOVA 검정 (5그룹 비교) 3) 사후검정: Tukey HSD로 어떤 요일 간 차이가 유의한지 확인 4) 메뉴 유형별 추가 분석 (탐색적)

    4. 시각화

    1) 요일별 만족도 막대그래프 + 오차 막대(표준편차) 2) 요일별 만족도 박스플롯 3) 한국어 라벨, 제목: '요일별 급식 만족도 비교'

    5. 해석 기준

    1) p < 0.05 → 요일 간 차이 유의 2) 평균 차이 0.5점 이상 → 실질적 차이 3) "이 분석은 특정 주의 데이터에 기반하므로 일반화에 한계가 있습니다" 명시

    6. Soul Document 작성 시 자주 하는 실수¶

    실수 1: 배경 맥락을 생략하는 경우

    증상: "데이터 분석해줘"로 시작 문제: AI가 방향 없이 모든 변수를 탐색, 의미 없는 결과 생성 해결: "왜 이 분석을 하는지, 결과를 어디에 쓸 것인지" 먼저 작성

    실수 2: 변수 설명 없이 파일만 첨부하는 경우

    증상: "첨부 파일 분석해줘" 문제: AI가 변수명을 추측, "grade"를 성적으로 오해 가능 해결: 주요 변수의 의미, 단위, 범위를 명시

    실수 3: 시각화를 요청하지 않는 경우

    증상: 분석만 요청, 그래프 없음 문제: 숫자만 나열된 결과, 패턴 파악 어려움 해결: 최소 1~2개의 그래프 유형과 축 라벨을 지정

    실수 4: 해석 기준 없이 '결론을 내려줘'라고 하는 경우

    증상: "결과를 해석해줘" 문제: AI가 "차이가 있습니다" 수준의 피상적 해석 해결: p-value 임계값, 실질적 차이 기준, 한계점 요구를 명시

    7. 데이터 리터러시 함정: 숫자가 거짓말하는 순간¶

    데이터를 다룰 때 가장 위험한 것은 "숫자니까 객관적이다"라는 착각입니다. 같은 데이터에서도 보는 방식에 따라 정반대의 결론이 나올 수 있습니다. 이 섹션에서는 데이터 분석에서 반드시 알아야 할 두 가지 함정을 인터랙티브 차트로 체험합니다.

    함정 1: 심슨의 역설 — 전체와 부분이 반대 결론을 말한다¶

    심슨의 역설(Simpson's Paradox)이란?

    전체 데이터를 합쳐서 보면 A라는 결론이 나오지만, 하위 그룹별로 나눠서 보면 정반대인 B라는 결론이 나오는 현상입니다. 이것은 통계적 오류가 아니라, 데이터에 숨겨진 교란 변수(confounding variable) 때문에 발생합니다.

    아래 차트를 직접 조작해 보세요. "전체 데이터"와 "학년별 분리" 버튼을 번갈아 눌러보면 같은 데이터에서 정반대 결론이 나오는 것을 확인할 수 있습니다.

    전체 데이터

    "스마트폰을 많이 쓸수록 성적이 높다"¶

    • 양의 상관관계 (r > 0)
    • 잘못된 결론: 스마트폰이 성적에 도움?
    • 교란 변수(학년)를 무시한 결과

    학년별 분리

    "학년 내에서는 반대: 스마트폰을 많이 쓸수록 성적이 낮다"¶

    • 각 학년 내 음의 상관관계 (r < 0)
    • 올바른 결론: 스마트폰 사용은 성적에 부정적
    • 3학년이 기본 성적과 스마트폰 사용 모두 높아서 전체를 왜곡

    Soul Document에 반영하기

    심슨의 역설을 방지하려면 Soul Document에 다음을 추가하세요:

    "분석 시 전체 데이터뿐만 아니라, 주요 하위 그룹(학년, 성별 등)별로 분리 분석도 수행하세요. 전체 결과와 하위 그룹 결과가 다를 경우, 교란 변수의 존재를 보고하세요."

    교사를 위한 심화: 왜 이런 일이 일어나는가?

    심슨의 역설은 교란 변수(confounding variable) 때문에 발생합니다.

    이 예시에서는 학년이 교란 변수입니다:

    • 3학년은 1학년보다 기본 성적이 높고 (학습 누적 효과)
    • 3학년은 1학년보다 스마트폰 사용시간도 길다 (자율성 증가)
    • 학년이라는 공통 원인이 두 변수를 동시에 높이므로, 전체를 합치면 양의 상관처럼 보입니다

    교실 활용법: "전체로 보니까 A인데, 그룹별로 보니까 B다. 어느 것이 진실인가?"라고 토론을 유도하세요. 정답은 "연구 질문에 따라 다르다"이지만, 대부분의 경우 하위 그룹 분석이 더 정확합니다.

    함정 2: 작은 표본의 함정 — 데이터가 적으면 상관계수도 불안정하다¶

    표본 크기가 작을 때의 문제

    데이터가 5~10개뿐일 때 계산한 상관계수는 매번 크게 달라집니다. 같은 모집단에서 표본을 다시 뽑으면 r = -0.80이 나올 수도, r = +0.20이 나올 수도 있습니다. 이것은 데이터 자체의 문제가 아니라, 표본이 모집단을 충분히 대표하지 못하기 때문입니다.

    아래 차트에서 표본 크기 슬라이더를 움직여 보세요. 작은 표본에서는 상관계수가 불안정하고, 큰 표본에서는 안정적으로 수렴하는 것을 확인할 수 있습니다.

    표본 크기 효과 — 데이터가 적으면 결론이 매번 달라진다
    표본 크기 상관계수 범위 (20회 반복) 해석
    5개 r = -0.90 ~ +0.40 결론 불가능 (방향조차 불안정)
    10개 r = -0.80 ~ -0.10 방향은 보이지만 강도 불안정
    30개 r = -0.70 ~ -0.35 대략적 경향 파악 가능
    100개 r = -0.60 ~ -0.45 안정적 결론 가능
    200개 r = -0.58 ~ -0.50 모집단 수준의 정확도

    Soul Document에 반영하기

    표본 크기의 함정을 방지하려면 Soul Document에 다음을 추가하세요:

    "분석 결과에 표본 크기(N)를 반드시 보고하세요. N < 30인 경우 '표본 크기가 작아 결론의 일반화에 한계가 있음'을 명시하세요. 가능하면 부트스트랩 신뢰구간을 제시하여 결과의 불확실성을 표현하세요."

    교사를 위한 심화: '30개 규칙'의 근거

    "표본 30개 이상이면 된다"는 흔한 경험적 기준이지만, 정확한 근거가 있습니다:

    • 중심극한정리: 표본 크기가 약 30 이상이면 표본 평균의 분포가 정규분포에 근사
    • 상관계수의 표준오차: SE(r) ≈ (1-r²)/√(n-2)이므로, n이 커질수록 SE가 급격히 감소
    • 실질적 기준: n=30일 때 |r| > 0.35가 p < 0.05를 만족 (양측 검정)

    단, 이는 최소 기준이며, 효과 크기가 작을 것으로 예상되면 더 큰 표본이 필요합니다. 학생들에게는 "최소 30개, 가능하면 100개 이상"으로 안내하세요.

    데이터 리터러시 핵심 정리¶

    함정 증상 해결
    심슨의 역설 전체 결과와 그룹별 결과가 정반대 Soul Document에 하위 그룹 분리 분석 요구
    작은 표본 상관계수가 매번 크게 달라짐 N ≥ 30 확보, 표본 한계 명시
    상관 ≠ 인과 "A가 B를 일으킨다"고 잘못 결론 "상관관계이며 인과 해석 불가" 명시
    이상치 영향 극단값 1~2개가 결론을 지배 이상치 포함/제외 두 결과 비교
    체리피킹 원하는 결론에 맞는 기간/변수만 선택 분석 전에 가설과 기간을 확정

    데이터 리터러시의 핵심 한 문장

    "숫자가 말하는 것"이 아니라 "숫자가 말하지 않는 것"을 찾아라.
