# "질문을 설계하라" — 데이터 분석의 근본 역량을 키우는 수업

---

## 📋 수업지도안

### 기본 정보

| 항목 | 내용 |
|------|------|
| 과목 | 정보 (2022 개정, 고등학교 일반선택) |
| 영역 | 데이터 |
| 주제 | **"현실의 모호한 문제를 데이터로 접근 가능한 질문으로 바꾸기"** |
| 시간 | 50분 (1차시) |
| 형태 | 2인 1조 |
| 준비물 | 학생 활동지 인쇄본, 상황카드 세트 (아래 참조), 화이트보드 마커 (있으면 좋음) |
| 사전 조건 | 데이터의 유형, 기초 통계 개념 학습 완료 |
| **특이사항** | **이 수업에서 컴퓨터는 사용하지 않습니다.** |

### 왜 이 수업인가

```
문제:
  기존 데이터 수업 → 데이터가 먼저 주어진다 → 학생은 "처리"만 한다
  AI 시대 수업    → AI에게 지시하는 법을 배운다 → 여전히 "도구 활용"이다

빠져 있는 것:
  현실에서는 데이터가 정리되어 주어지지 않는다.
  "우리 학교에 문제가 있는 것 같아" 같은 모호한 상황에서
  "무엇을 어떤 데이터로 확인할 수 있는가"를 만들어내는 능력.

  이것이 데이터 분석의 진짜 첫 번째 단계이며,
  AI가 아무리 발전해도 대체할 수 없는 영역이다.
```

### 학습목표

1. 모호한 현실 상황을 **데이터로 답할 수 있는 구체적 질문**으로 변환할 수 있다.
2. 질문에 답하기 위해 **어떤 데이터가 필요한지** 역으로 설계할 수 있다.
3. "좋은 분석 질문"과 "나쁜 분석 질문"의 차이를 **자신의 언어로 설명**할 수 있다.

### 성취기준 연결

| 2022 개정 성취기준 방향 | 이 수업에서의 구현 |
|---|---|
| 데이터를 수집·분석·시각화한다 | 분석에 앞서 "무엇을 수집해야 하는가"를 설계하는 역량 |
| 실생활 문제를 발견하고 해결한다 | 모호한 현실 상황에서 데이터로 접근 가능한 문제를 발견 |
| 컴퓨팅 사고력 | 추상화(모호함→구체적 질문), 분해(큰 문제→작은 질문) |

### 50분 타임라인

```
[도입: 7분] 데이터 분석은 어디서 시작하는가
├─ 00-03분: "급식이 맛없다" — 이걸 어떻게 확인하지?
└─ 03-07분: 모호한 불만 → 분석 가능한 질문 시연

[활동 1: 18분] 상황카드 → 질문 3단 변환
├─ 07-10분: 상황카드 배부 + 활동 설명
├─ 10-20분: 2인 1조 — 모호한 상황을 질문으로 변환 (활동지 Part A)
└─ 20-25분: 조별 발표 + 전체 피드백

[활동 2: 15분] 질문에서 데이터 역설계
├─ 25-28분: "이 질문에 답하려면 뭘 모아야 하지?"
├─ 28-37분: 데이터 수집 설계서 작성 (활동지 Part B)
└─ 37-40분: 조 간 교차 검토

[정리: 10분] 좋은 질문의 조건
├─ 40-45분: 전체 토론 — 좋은 질문 vs 나쁜 질문
└─ 45-50분: 성찰 + 다음 차시 안내 (활동지 Part C)
```

### 상세 수업 진행

#### 도입 (00-07분): 데이터 분석은 어디서 시작하는가

**[강의 스크립트: 00-03분]**

선생님: "데이터 분석 수업을 하면, 보통 이렇게 시작하죠. '자, 여기 데이터가 있습니다. 이걸 분석하세요.' 맞죠?"

(학생들 고개 끄덕)

선생님: "근데 현실에서는 데이터가 깔끔하게 주어지지 않아요. 이런 식으로 시작됩니다."

(화면에 크게)

```
"급식이 맛없다는 말이 많은데, 
 진짜 문제가 뭔지 모르겠어."
```

선생님: "이건 분석할 수 있는 문장인가요?"

(학생: "아니요", "뭘 분석하라는 건지 모르겠는데")

선생님: "맞아요. 이건 '느낌'이지 '질문'이 아닙니다. 데이터 분석의 진짜 첫 번째 단계는 코드를 짜는 것도 아니고, AI에게 지시하는 것도 아닙니다. **모호한 느낌을 데이터로 답할 수 있는 질문으로 바꾸는 것**입니다."

**[시연: 03-07분]**

선생님: "같이 해봅시다. '급식이 맛없다는 말이 많다' — 이걸 분석 가능한 질문으로 바꿔보면?"

(학생 반응을 유도하며 칠판에 정리)

```
[원래 상황]
  "급식이 맛없다는 말이 많은데, 진짜 문제가 뭔지 모르겠어."

         ↓ 1단계: "모호한 느낌"을 "구체적 현상"으로

[현상 질문] 
  "급식에 대한 불만이 실제로 얼마나 많은가?"
  "어떤 종류의 불만이 가장 많은가? 맛? 양? 메뉴 다양성?"

         ↓ 2단계: "현상"을 "원인 추적 가능한 질문"으로

[분석 질문]
  "요일별로 만족도 차이가 있는가? 
   (특정 요일 메뉴에 문제가 있는 건 아닌가?)"
  "학년별로 불만 유형이 다른가?
   (1학년은 양, 3학년은 맛이 문제라면 대응이 달라야)"

         ↓ 3단계: "질문"에서 "의사결정 연결"로

[의사결정 연결]
  "이 분석 결과에 따라 무엇을 바꿀 수 있는가?"
  → 요일별 차이 → 특정 요일 메뉴 교체
  → 학년별 차이 → 학년별 선택 메뉴제 도입 검토
```

선생님: "보이시죠? '급식이 맛없다'가 세 단계를 거치면 '요일별 만족도 차이가 있는가?'라는 데이터로 답할 수 있는 질문이 됩니다. 그리고 이 질문의 답에 따라 뭘 할지까지 연결됩니다."

선생님: "이 변환 능력 — 이건 AI가 대신 못 해요. AI한테 '급식이 맛없대, 분석해줘'라고 하면 뭐가 나올까요? 아까 봤던 것처럼 쓸데없는 평균값 나열이 나옵니다. **좋은 질문을 먼저 만들어야 좋은 분석이 나옵니다.**"

#### 활동 1 (07-25분): 상황카드 → 질문 3단 변환

**[활동 안내: 07-10분]**

선생님: "이제 여러분 차례입니다. 조별로 상황카드를 1장씩 나눠드릴게요."

(상황카드 배부)

선생님: "활동지 Part A를 보세요. 아까 제가 '급식' 예시에서 했던 걸 여러분이 직접 합니다. 3단계 변환이에요."

```
활동지 Part A: 질문 3단 변환

여러분의 상황카드: "____________________________"

[1단계] 모호한 느낌 → 구체적 현상
  이 상황에서 확인해야 할 '현상'은 정확히 무엇인가?
  → (여기에 작성)

[2단계] 구체적 현상 → 분석 가능한 질문
  데이터로 답할 수 있는 질문으로 바꾸면?
  (최소 2개, 최대 4개)
  → 질문 1:
  → 질문 2:
  → (추가 질문:)

[3단계] 질문 → 의사결정 연결
  이 질문의 답이 나오면 무엇을 결정/변경할 수 있는가?
  → 질문 1의 답에 따라:
  → 질문 2의 답에 따라:
```

선생님: "여기서 핵심 규칙 하나. **2단계에서 만드는 질문은 반드시 '예/아니오' 또는 '숫자'로 답할 수 있어야 합니다.** '급식이 왜 맛없을까?'는 안 됩니다 — 너무 열린 질문이에요. '요일별 만족도에 차이가 있는가?'는 됩니다 — 데이터로 확인 가능하니까."

선생님: "10분 드립니다. 짝이랑 같이 하세요."

**[개인/조별 작성: 10-20분]**

(순회 지도)

**[수업 장면: 활동 1]**

--- 상황카드 ① "학생들이 잠을 못 자는 것 같다" ---

민수/지훈 조:
민수: "1단계에 뭐라고 쓰지? 그냥 잠을 못 잔다는 거잖아."
지훈: "'못 잔다'가 실제로 얼마나인지 확인하는 게 1단계 아닐까?"
민수: "아... '우리 학교 학생의 평균 수면시간은 실제로 몇 시간인가?' 이런 거?"
선생님: (다가가며) "좋아요. 근데 한 가지 더 — '못 잔다'의 기준이 뭐예요? 몇 시간 미만이면 '못 자는' 건가요?"
민수: "권장 수면시간이 있잖아요. 8시간?"
선생님: "좋아. 그러면 1단계를 '우리 학교 학생 중 권장 수면시간(8시간) 미만인 비율은 얼마인가?'로 구체화할 수 있겠죠. '느낌'이 '측정 가능한 현상'이 됐어요."
지훈: "2단계는... '학년별로 수면시간 차이가 있는가?' 이런 건가?"
선생님: "좋아요. 또 뭐가 있을까? 수면 부족의 원인을 추적할 수 있는 질문은?"
민수: "밤에 뭐 하느라 못 자는지? 근데 그건 데이터가..."
선생님: "바로 그 지점이야. '데이터로 답할 수 있는가?'를 스스로 점검하고 있네. 설문으로 물을 수 있는 항목이면 데이터가 될 수 있겠지?"
지훈: "'취침 전 스마트폰 사용시간과 수면시간의 관계'면 되나?"
선생님: "됩니다. 그게 바로 분석 가능한 질문이에요."

--- 상황카드 ③ "급식 만족도가 낮다" ---

수연/하윤 조:
수연: "급식은 선생님이 예시로 해주셨으니까 쉽겠다."
하윤: "근데 선생님 예시 그대로 쓰면 안 될 것 같은데..."
수연: "그러면 다른 각도에서. '만족도가 낮다'가 진짜인지부터. 전체적으로 낮은 건지 특정 메뉴만 낮은 건지."
하윤: "아, 그러면 2단계 질문이 '전체 평균 만족도 vs 메뉴별 만족도에 차이가 있는가?' 이렇게?"
선생님: (지나가며) "좋은데, 3단계도 생각해봐. 이 답이 나오면 뭘 바꿀 수 있어?"
수연: "메뉴별로 차이가 크면, 만족도 낮은 메뉴를 교체하면 되니까..."
하윤: "근데 만족도 높은 메뉴만 하면 영양 균형이..."
선생님: "그 긴장감이 진짜 데이터 분석이에요. 분석 결과를 무조건 따르는 게 아니라, 다른 가치와 균형을 잡아야 하잖아. 3단계에 그것도 적어봐."

**[조별 발표: 20-25분]**

선생님: "2-3조만 발표해주세요. 핵심만 — 원래 상황, 2단계 질문, 3단계 의사결정 연결. 30초씩."

(발표 후)

선생님: "발표 들으면서 느낀 점 있어요? 같은 상황카드를 받은 조도 질문이 다를 수 있어요. 그게 정상입니다. **같은 현실에서 다른 질문을 만드는 것** — 이게 설계입니다. 정답은 없어요. 다만 좋은 질문과 덜 좋은 질문의 차이는 있습니다."

#### 활동 2 (25-40분): 질문에서 데이터 역설계

선생님: "좋은 질문을 만들었으면, 이제 거꾸로 갑니다. '이 질문에 답하려면, 어떤 데이터를 어떻게 모아야 하지?'"

선생님: "이게 왜 중요하냐면 — 보통 데이터 수업에서는 데이터가 미리 주어져요. 실제로는 여러분이 '뭘 모을지'를 먼저 결정해야 해요. 활동지 Part B를 보세요."

| [1]-[2] 데이터 항목+유형 | 12점 | 질문에 정확히 대응하는 항목 3개 이상 + 유형 명시(12) / 부분 대응(7) / 무관한 항목(3) |
| [3] 대상 설계 | 6점 | 대상과 인원을 이유와 함께 제시(6) / 이유 없이 제시(3) / 미작성(0) |
| [4] 방법 설계 | 7점 | 방법 + 장단점 구체적(7) / 방법만 제시(4) / 미작성(0) |
| [5] 함정 발견 | 10점 | 데이터 수집의 실질적 한계를 구체적으로 인식(10) / 표면적 한계(5) / 미작성(0) |

### 생기부 기재 예시

```
[상위 수준]
데이터 분석 설계 실습에서 '학생 수면 부족' 상황을 분석 가능한 질문으로
변환하는 과정에서, '못 잔다'는 모호한 표현을 '권장 수면시간 미달 비율'이라는
측정 가능한 지표로 구체화함. '학년별 수면시간 차이가 있는가'와 '취침 전
스마트폰 사용시간과 수면시간의 상관관계'라는 분석 질문을 도출하고, 각 질문의
답에 따른 학교 차원의 대응 방안까지 연결함. 데이터 역설계 과정에서 자기 보고식
설문의 과소 보고 가능성을 자발적으로 지적하며, 데이터 수집 방법론의 한계를
인식하는 비판적 사고를 보여줌.

[중위 수준]
데이터 분석 설계 실습에서 모호한 현실 상황을 데이터로 답할 수 있는 구체적
질문으로 변환하는 과정을 경험함. 분석 질문에 필요한 데이터 항목과 수집
대상을 설계하는 과정에서 질문의 구체성이 데이터 설계의 품질을 결정한다는
것을 이해함.
```

## 2차시 연결 안내

```
1차시 (이 수업):  질문 설계 + 데이터 역설계  ← 컴퓨터 없음
2차시 (다음):     설계한 질문으로 AI에게 분석 시키기 + 결과 검증

1차시에서 만든 활동지 Part A의 질문과 Part B의 데이터 설계가
2차시의 "분석 지침서(Soul Document)"의 입력이 됩니다.

1차시의 질문이 좋을수록 → 2차시의 AI 분석 결과가 좋아진다.
이 연결을 학생이 직접 경험하는 것이 핵심.
```
