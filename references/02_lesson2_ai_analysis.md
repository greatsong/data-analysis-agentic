# 데이터 분석 수업의 에이전틱 전환: 50분 1차시 수업 세트

---

## 📋 수업지도안

### 기본 정보

| 항목 | 내용 |
|------|------|
| 과목 | 정보 (2022 개정, 고등학교 일반선택) |
| 영역 | 데이터 |
| 단원 | 데이터 수집·분석·시각화 |
| 주제 | **"AI에게 데이터 분석을 시켜보자 — 좋은 지시가 좋은 분석을 만든다"** |
| 시간 | 50분 (1차시) |
| 형태 | 개인 실습 (2인 1조도 가능) |
| 준비물 | 학생 1인당 PC 또는 노트북, Claude/ChatGPT 등 LLM 접근 가능 환경, 학생 활동지 인쇄본 |
| 사전 조건 | 데이터의 유형(정형/비정형), 평균·중앙값·표준편차 기초 개념 학습 완료 |

### 학습목표

1. AI에게 데이터 분석을 요청하는 **구조화된 지침서**를 작성할 수 있다.
2. AI가 수행한 분석 결과의 **타당성과 한계**를 평가할 수 있다.
3. 지침서를 수정하여 분석 품질을 **반복적으로 개선**할 수 있다.

### 성취기준 연결

| 2022 개정 성취기준 방향 | 이 수업에서의 구현 |
|---|---|
| 데이터를 수집·분석·시각화한다 | AI에게 분석·시각화를 지시하고 결과를 검증 |
| 데이터의 의미를 해석한다 | AI 분석 결과를 비판적으로 해석하고 오류 발견 |
| 컴퓨팅을 통한 문제 해결 | 문제 정의 → AI 지시 → 결과 검증 → 지침 수정 루프 |

### 이 수업의 핵심 전환

```
기존 수업: 선생님이 데이터 제공 → 학생이 판다스/엑셀로 직접 분석 → 결과 제출
이 수업:   학생이 분석 목적 정의 → AI에게 분석 지시 → 결과 검증 → 지시 개선

바뀌는 것: 분석 "도구 조작"에서 분석 "설계와 검증"으로 초점 이동
안 바뀌는 것: 데이터 해석 능력, 통계 개념 이해 — 이것은 오히려 더 깊어짐
```

### 50분 타임라인

```
[도입: 8분] 왜 AI에게 시키는데 더 어려운가
├─ 00-03분: 도입 질문 — "AI한테 시키면 끝 아닌가요?"
└─ 03-08분: 시연 — 같은 데이터, 나쁜 지시 vs 좋은 지시

[활동 1: 15분] 분석 지침서(Soul Document) 작성
├─ 08-12분: 지침서 양식 설명 + 데이터 소개
└─ 12-23분: 개인별 지침서 작성 (활동지 Part A)

[활동 2: 15분] AI 실행 + 결과 검증
├─ 23-30분: AI에게 지침서 입력 → 분석 결과 받기
└─ 30-38분: 결과 검증 체크리스트 작성 (활동지 Part B)

[활동 3: 7분] 지침서 수정 → 재실행
├─ 38-42분: 지침서 수정 (활동지 Part C)
└─ 42-45분: 재실행 → 개선 확인

[정리: 5분] 오늘 배운 것
└─ 45-50분: 핵심 정리 + 성찰 (활동지 Part D)
```

### 상세 수업 진행

#### 도입 (00-08분): 왜 AI에게 시키는데 더 어려운가

**[강의 스크립트: 00-03분]**

선생님: "오늘 수업에서는 데이터 분석을 합니다. 근데 여러분이 직접 코드를 짜지는 않을 거예요. AI한테 시킬 겁니다."

(학생 반응: "오 쉽겠다", "그러면 우리는 뭐 하는 거예요?")

선생님: "그렇죠? 쉬울 것 같죠? 그런데 하나 보여드릴게요."

**[시연: 03-08분 — 나쁜 지시 vs 좋은 지시]**

선생님이 화면을 띄우고, 같은 데이터(서울시 월별 평균 기온 데이터 등 단순한 공개 데이터)에 대해 두 가지 지시를 AI에게 보냅니다.

```
[나쁜 지시]
"이 데이터 분석해줘"

→ AI 결과: 평균, 최대, 최소 등 기계적 통계 나열.
  시각화는 단순 꺾은선 그래프 1개.
  해석은 뻔한 문장. ("여름이 가장 덥고 겨울이 가장 춥습니다.")
```

선생님: "자, 이게 쓸 만한가요? 틀린 건 없는데... 그래서 뭐? 라는 느낌이죠."

```
[좋은 지시]
"서울시 월별 평균 기온 데이터를 분석해줘.
 목적: 최근 10년간 여름 폭염 심화 추세가 있는지 확인
 분석 요청:
 1. 연도별 7-8월 평균 기온 추이 계산
 2. 최근 5년 vs 이전 5년의 7-8월 평균 기온 비교 (t-test)
 3. 연도별 '폭염일수(일 최고기온 33도 이상)' 추이
 출력 형태: 추이 그래프 + 통계 요약 + 한계점 명시"

→ AI 결과: 목적에 맞는 분석, 통계 검정, 의미 있는 시각화,
  한계점까지 스스로 명시.
```

선생님: "같은 AI, 같은 데이터인데 결과가 완전히 다르죠. 차이가 뭐예요?"

(학생: "질문이 다르니까요", "두 번째가 구체적이니까요")

선생님: "맞아요. AI 시대에 데이터 분석 능력이란 — 판다스 코드를 잘 치는 게 아니라 **'무엇을 왜 어떻게 분석할지'를 설계하는 능력**입니다. 오늘 이걸 연습합니다."

**[예상 Q&A]**

Q: "그러면 판다스 같은 건 안 배워도 되나요?"
A: "코드를 이해하는 능력은 여전히 필요해요. AI가 만든 분석이 맞는지 확인하려면 코드를 읽을 줄 알아야 하거든요. 다만 오늘은 '설계와 검증'에 집중합니다."

Q: "두 번째 지시를 쓰려면 통계를 알아야 하잖아요?"
A: "정확합니다! 그게 핵심이에요. AI에게 좋은 지시를 내리려면 데이터 분석에 대한 이해가 오히려 더 깊어야 합니다. AI가 해주는 건 실행이지, 설계가 아닙니다."

#### 활동 1 (08-23분): 분석 지침서(Soul Document) 작성

**[실습 안내 스크립트: 08-12분]**

선생님: "자, 이제 여러분이 직접 해봅시다. 활동지 Part A를 펼쳐주세요."

선생님: "오늘 분석할 데이터를 소개할게요."

(화면에 데이터 소개)

```
📊 오늘의 데이터: 우리 학교 학생 200명 설문 결과
(가상 데이터, 사전 준비하여 CSV로 제공)

항목: 학년, 성별, 하루 스마트폰 사용시간(시간),
      하루 수면시간(시간), 주관적 피로도(1-10),
      주관적 학업만족도(1-10), 운동 횟수(주당)

※ 실제 수업에서는 학교 실정에 맞는 데이터로 교체 가능
  (급식 만족도, 통학시간, 동아리 활동 등)
```

선생님: "이 데이터를 가지고 AI에게 분석을 시킬 건데, 바로 시키지 않습니다. 먼저 '분석 지침서'를 씁니다. 에이전틱 워크플로우에서는 이걸 Soul Document라고 불러요. AI가 일을 시작하기 전에 읽는 설계서입니다."

선생님: "활동지에 5칸이 있어요. 하나씩 채워봅시다."

```
분석 지침서 5칸:
  ① 분석 질문: 이 데이터로 알고 싶은 것은? (1문장)
  ② 분석 이유: 왜 이것이 궁금한가? (1문장)
  ③ 분석 방법: AI에게 어떤 분석을 요청할 것인가? (구체적으로)
  ④ 출력 형태: 어떤 형태로 결과를 받고 싶은가?
  ⑤ 주의사항: AI가 실수할 수 있는 부분은?
```

선생님: "예시를 하나 보여드릴게요."

```
[예시]
① 분석 질문: 스마트폰 사용시간이 긴 학생일수록 수면시간이 짧은가?
② 분석 이유: 학교에서 수면 캠페인을 하려는데, 스마트폰이 원인인지 확인하고 싶다.
③ 분석 방법:
   - 스마트폰 사용시간과 수면시간의 산점도 + 상관계수 계산
   - 학년별로 나눠서도 분석 (학년에 따라 패턴이 다를 수 있으니)
   - 사용시간 구간별(2시간 미만/2-4시간/4시간 이상) 수면시간 평균 비교
④ 출력 형태: 산점도 그래프 + 상관계수 표 + 구간별 비교 표 + 해석 요약
⑤ 주의사항: 상관관계를 인과관계로 오해하지 않도록 주의 문구 포함할 것
```

선생님: "⑤번을 보세요. '상관관계 ≠ 인과관계' — 이걸 미리 적어놓으면 AI가 결과에서 '스마트폰이 수면 부족의 원인'이라고 단정하는 실수를 줄일 수 있어요. 이게 설계의 힘입니다."

**[개인 작성: 12-23분]**

선생님: "이제 11분 드릴 테니 활동지 Part A를 채워주세요. 분석 질문은 자유입니다. 이 데이터에서 여러분이 진짜 궁금한 걸 질문으로 만들어보세요."

(순회 지도 — 학생이 막히는 포인트)

**[수업 장면: 활동 1]**

민수: (③번에서 멈춤) "분석 방법을 뭐라고 써야 할지 모르겠어요. 그냥 '분석해줘'라고 하면 안 되나요?"
선생님: "아까 시연 기억나? '분석해줘'가 나쁜 지시였잖아. 네가 궁금한 게 뭔데?"
민수: "운동을 많이 하는 애들이 피로도가 낮은지 궁금해요."
선생님: "좋아. 그러면 어떤 분석이 필요할까? 운동 횟수와 피로도 사이의 관계를 보려면?"
민수: "아... 그것도 산점도? 아니면 운동 횟수별로 나눠서 평균 비교?"
선생님: "둘 다 가능하지. 둘 다 써. AI에게는 구체적일수록 좋아."

수연: (⑤번을 비워둠)
선생님: "수연이는 주의사항 안 썼네. AI가 이 분석에서 실수할 수 있는 게 뭘까?"
수연: "음... 데이터가 200명밖에 안 돼서 일반화하면 안 되는 거?"
선생님: "바로 그거! '표본 크기가 작으므로 일반화에 주의'라고 써놔. 이런 걸 적는 능력이 데이터 리터러시야."

#### 활동 2 (23-38분): AI 실행 + 결과 검증

**[실습 안내 스크립트: 23-30분]**

선생님: "지침서 완성됐으면, 이제 AI에게 보냅시다. Claude나 ChatGPT를 열고, 데이터 파일을 첨부한 다음, 여러분이 쓴 지침서 ①~⑤를 그대로 복사해서 보내세요."

선생님: "결과가 나오면 — 바로 만족하지 마세요. 활동지 Part B를 펼치세요. '결과 검증 체크리스트'가 있습니다."

(학생들이 AI에 지침서 입력, 결과 받기: 약 5분)

**[결과 검증: 30-38분]**

선생님: "자, AI가 결과를 줬죠? 이제부터가 진짜입니다. Part B의 체크리스트를 하나씩 확인하세요."

```
결과 검증 체크리스트 (Part B):

□ ① 내가 요청한 분석을 빠짐없이 수행했는가?
     → 요청 항목 대비 실제 수행 항목 체크

□ ② 숫자가 말이 되는가?
     → 평균값, 범위, 비율 등이 상식적으로 타당한가?
     → 예: "평균 수면시간 2시간" → 뭔가 잘못됨

□ ③ 시각화가 데이터를 정확히 반영하는가?
     → 축 라벨, 범위, 단위가 맞는가?
     → 그래프가 주장을 뒷받침하는가?

□ ④ AI의 해석에 논리적 비약은 없는가?
     → 상관관계를 인과관계로 단정하지 않았는가?
     → 결론이 데이터 범위를 넘어서 일반화하지 않았는가?

□ ⑤ 내가 ⑤번에 적은 주의사항을 AI가 반영했는가?

→ 발견한 문제점을 아래에 구체적으로 적으세요.
```

선생님: "중요한 건 — AI가 틀린 걸 발견하는 게 오늘 수업의 성공입니다. AI가 완벽한 답을 줬다면 오히려 의심해야 해요."

**[수업 장면: 활동 2]**

지훈: "선생님, AI가 상관계수 0.73이라고 했는데 '강한 양의 상관관계'래요. 이거 맞아요?"
선생님: "0.73이면 어떤 수준이지?"
지훈: "음... 0.7 이상이면 강한 거 맞는 것 같은데..."
선생님: "좋아. 그런데 산점도를 봐봐. 데이터가 실제로 선형으로 보여?"
지훈: (산점도 확인) "아... 이상치가 몇 개 있는데 이거 때문에 상관계수가 높아진 것 같아요."
선생님: "바로 그거야. 숫자만 보면 놓치는 거를 시각화를 통해 발견한 거지. 그걸 Part B에 적어."

하윤: "AI가 제가 요청 안 한 분석도 추가로 해줬어요. 이것도 쓸까요?"
선생님: "어떤 분석인데?"
하윤: "성별에 따른 차이 분석이요."
선생님: "그건 네 질문과 관련이 있어?"
하윤: "음... 직접적으로는 아닌데 흥미로운 결과가 나왔어요."
선생님: "좋아. 그건 '추가 발견'으로 기록하되, 네 원래 분석 질문에 대한 답이 되는지 아닌지는 구분해야 해. AI가 준 걸 다 받아들이면 분석의 초점이 흐려져."

---

#### 활동 3 (38-45분): 지침서 수정 → 재실행

**[실습 안내 스크립트: 38-42분]**

선생님: "Part B에서 문제를 발견했죠? 이제 Part C입니다. 지침서를 수정하세요."

선생님: "핵심 원칙 하나 — **결과물을 직접 고치지 마세요. 지침서를 고치세요.** AI가 만든 그래프를 직접 편집하는 게 아니라, 지침서의 ③번이나 ④번을 수정해서 다시 시키는 겁니다. 이게 에이전틱 워크플로우의 핵심이에요."

```
Part C: 지침서 수정

수정 전 (원래 지침서에서 부족했던 부분):
→ _______________

수정 후 (어떻게 고쳤는가):
```
