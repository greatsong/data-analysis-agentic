# 데이터 분석 에이전틱 수업 3~4차시 패키지

> **시리즈 전체 구조**
> 1차시: 질문 설계 + 데이터 역설계 (컴퓨터 없음)
> 2차시: AI에게 분석 시키기 + 결과 검증 + 지침 수정
> **3차시: 동료 피어리뷰 — "남의 분석을 부숴라"** ← 이 문서
> **4차시: 자유 주제 미니 프로젝트 — "처음부터 끝까지 혼자 돌려라"** ← 이 문서

---

# 3차시: 동료 피어리뷰 — "남의 분석을 부숴라"

## 📋 수업지도안

### 기본 정보

| 항목 | 내용 |
|------|------|
| 과목 | 정보 (2022 개정, 고등학교 일반선택) |
| 영역 | 데이터 |
| 주제 | **"다른 조의 분석을 비판적으로 검토하고, 내 분석도 검토받기"** |
| 시간 | 50분 (1차시) |
| 형태 | 2인 1조 (1-2차시와 동일 조) |
| 준비물 | 1-2차시 활동지 (각 조의 완성본), 3차시 활동지, PC (AI 재실행용) |
| 사전 조건 | 1-2차시 완료. 각 조가 질문 설계 + AI 분석 결과 + 검증 기록을 보유 |

### 학습목표

1. 다른 조의 분석에서 **질문 설계, 데이터 설계, 분석 결과의 약점**을 구체적으로 찾을 수 있다.
2. 비판적 피드백을 받고 자기 분석의 약점을 **인정하고 개선 방향을 제시**할 수 있다.
3. "좋은 분석"의 기준을 **동료와의 비교를 통해** 자기 언어로 정립할 수 있다.

### 이 수업의 핵심

```
1-2차시에서 학생은 자기 질문, 자기 분석에 몰입했다.
자기 작업물에는 눈에 안 보이는 약점이 있다.
남의 분석을 볼 때 비로소 비판적 시선이 작동한다.

그리고 남의 분석을 깨는 경험이
자기 분석을 깨는 능력으로 전이된다.
```

### 50분 타임라인

```
[도입: 5분] 왜 남의 분석을 봐야 하는가
└─ 00-05분: "자기 글의 오탈자는 안 보인다" 비유

[활동 1: 20분] 공격 라운드 — 남의 분석 부수기
├─ 05-08분: 교환 규칙 + 검토 프레임 설명
├─ 08-20분: 상대 조 분석 전체를 검토 (활동지 Part A)
└─ 20-25분: 피드백 전달 + 질의응답 (조 대 조)

[활동 2: 15분] 수비 라운드 — 내 분석 방어 또는 수용
├─ 25-30분: 받은 피드백 분류 — 수용 vs 반론 (활동지 Part B)
└─ 30-40분: 핵심 약점 1개 선택 → AI로 재분석 (활동지 Part C)

[정리: 10분] "좋은 분석"이란 무엇인가
├─ 40-46분: 전체 토론 — 검토하면서 발견한 패턴
└─ 46-50분: 성찰 (활동지 Part D)
```

(학생: "안 보여요")

선생님: "규칙 하나. 오늘의 목표는 **상대 조의 분석에서 약점을 최대한 많이 찾는 것**입니다. '잘했어요'는 쓸 필요 없어요. 날카로울수록 좋습니다. 대신 — 근거 없이 '별로다'는 안 됩니다. **구체적으로 어디가 왜 약한지** 지적해야 합니다."

#### 정리 (40-50분): "좋은 분석"이란 무엇인가

**[전체 토론: 40-46분]**

선생님: "오늘 남의 분석을 검토하면서, '아 이건 잘했네' 또는 '이건 문제다'라고 느낀 순간이 있었을 거예요. 거기서 패턴을 뽑아봅시다."

선생님: "좋은 분석의 조건 — 1차시에서 한 번 정리했죠. 오늘 추가되는 게 있나요?"

(학생 발언을 칠판에 추가)

```
1차시에서 발견한 조건:
  ✓ 질문이 데이터로 답할 수 있다
  ✓ 질문이 구체적이다
  ✓ 의사결정에 연결된다
  ✓ 데이터를 실제로 모을 수 있다

3차시에서 추가된 조건:
  ✓ 검증이 형식적이지 않다 — 근거가 있다
  ✓ 데이터의 한계를 인정하고 명시한다
  ✓ 피드백을 받으면 개선할 수 있다 — 설계가 수정 가능하다
  ✓ 다른 사람이 봐도 논리가 통한다
```

### 평가 루브릭 (3차시)

| 평가 요소 | 배점 | 상 (A) | 중 (B) | 하 (C) |
|---|---|---|---|---|
| **공격: 검토의 질** (Part A) | 35점 | 세 렌즈 모두에서 구체적 약점 발견 + 대안 제안 | 2개 렌즈에서 약점 발견, 대안은 약함 | 1개 이하, "잘했다" 위주이거나 근거 없는 비판 |
| **수비: 수용/반론의 질** (Part B) | 30점 | 수용에 자기 언어의 근거, 또는 반론에 설득력 있는 논리 | 수용/반론은 했으나 근거가 부족 | "인정합니다" 또는 "아닌데요" 수준 |
| **개선 실행** (Part C) | 20점 | 약점→지침 수정→재분석으로 실제 개선 확인 | 지침 수정은 했으나 개선 확인이 약함 | 미실행 또는 형식적 |
| **성찰** (Part D) | 15점 | 자기 시선의 편향을 구체적으로 인식 | 피상적 성찰 | 미작성 |

### 생기부 기재 예시 (3차시)

```
[상위 수준]
동료 피어리뷰 활동에서 상대 조의 데이터 분석을 세 가지 관점(질문의 질,
데이터 설계 타당성, AI 분석 검증 충실도)으로 체계적으로 검토함. 상대 조의
자기 보고식 설문 설계에서 응답 편향 가능성을 지적하고, 익명 처리 시
비교 분석 제약이라는 트레이드오프를 구체적으로 제시함. 자기 분석에 대한
피드백을 수용하여 분석 지침서에 데이터 한계 명시 조건을 추가하고 AI 재분석을
실행하여 결과의 해석 타당성을 실제로 개선함.
```

# 4차시: 자유 주제 미니 프로젝트 — "처음부터 끝까지 혼자 돌려라"

## 📋 수업지도안

### 기본 정보

| 항목 | 내용 |
|------|------|
| 과목 | 정보 (2022 개정, 고등학교 일반선택) |
| 영역 | 데이터 |
| 주제 | **"자유 주제로 질문 설계→데이터 설계→AI 분석→검증→발표"** |
| 시간 | 50분 (1차시) |
| 형태 | 개인 (원하면 2인 1조 허용) |
| 준비물 | PC, AI 도구 접근, 4차시 활동지, 공개 데이터 목록 (아래 참조) |
| 사전 조건 | 1~3차시 완료 |

### 학습목표

1. 자기가 관심 있는 현실 문제에서 출발하여, **전체 분석 사이클을 독립적으로** 수행할 수 있다.
2. 분석 결과를 바탕으로 **근거 있는 제안**을 1분 이내로 발표할 수 있다.
3. 1~3차시에서 배운 기준으로 **자기 분석의 한계를 스스로 명시**할 수 있다.

### 이 수업의 핵심

```
1차시: 교사가 상황카드를 줬다 → 주제가 주어짐
2차시: 교사가 데이터를 줬다 → 재료가 주어짐
3차시: 교사가 검토 프레임을 줬다 → 구조가 주어짐

4차시: 아무것도 주어지지 않는다.
학생이 스스로:
  - 주제를 정하고
  - 질문을 만들고
  - 데이터를 찾고
  - AI에게 시키고
  - 검증하고
  - 한계를 명시하고
  - 발표한다.

1~3차시의 스캐폴딩을 제거한 상태에서
학생이 독립적으로 사이클을 돌릴 수 있는가 — 이것이 최종 평가.
```
